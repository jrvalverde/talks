10 APPENDIX II: GCG LIST FILES

A list file is a text file of sequence file names or database references. It is an excellent and flexible way for you to produce a specific mini-database, e.g. to do fasta searches against or look for patterns in.Each entry is on a separate line. These can be sequences either in your Unix account, in a GCG database or even in another list file. e.g. a trivial example:initial comments.. ! A GCG format thing (anything above this is a comment)em:hsfau ! an entry in the embl databaserabbit.seq ! a file in your directoryem_rodent:mm* ! all rodent entries in the EMBL full release.! em:hsaug this entry is commented out.rodent:* ! the GCG rodent taxonomic category@hedgehog.list ! abother list file called “hedgehog.list”Many GCG programs can generate list files e.g. :
  • lookup (searches for keywords)
  • fasta -names (search sequence against sequence databases)
  • findpatterns-names (searches for a pattern in the databases)
You can use a text editor like pico or xemacs, to create and edit your list file. You can make use of simple Unix commands to add the sequences in your directory to a list file:
  • Create a list file: ls *.seq > new.list
  • Add sequences to an existing file: ls ins*.seq >> insulin.list
  • Or from another directory: ls /people/nobel/*.seq > prize.list
The @ symbol in front of your list file tells GCG programs that the file is a file of filenames and not a sequence. e.g.
  • Search your query sequence against: fasta query.seq @insulin.list
  • Multiply align your sequences: pileup @insulin.list
  • Compare all sequences: polydot @insulin.list
  • Select sequences with patterns: findpatterns @insulin.list
