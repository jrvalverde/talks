3 THE UNIX OPERATING SYSTEM

3.1 THE UNIX OPERATING SYSTE M GENERALITIES


Unix is an operating system, Windows NT is another example of an operating system (OS). The job of the operating system is to sit between the user(s) and the computer hardware, so that the computer hardware details are shielded from the user. Other software on the computer communicates to the hardware using the operating system as an interface. This makes program writing much easier.
Unix is a well established, very widespread operating system. It is command line driven, with a huge number of often terse, but powerful commands. It is used on many of the powerful computers at bioinformatics centres and also on many workstations. In contrast to Windows, it is designed to allow many users to run their programs simultaneously on the same computer. Another advantage is that it is designed to work in networks (for example, most of the Internet is Unix based).
The purpose of this practical is to teach you the few Unix commands you will need to know to:
  • Run the many bioinformatics programs which require the Unix command line.
  • View and manipulate your input and output files.
Spending a couple of hours learning a few Unix concepts and commands, will certainly make your use of our and other facilities more productive. The only commands that you really need to know in UNIX are:
  1. ls list files
  2. more display contents of file on screen
  3. cp make a new copy of a file
  4. rm remove (delete) a file
  5. mv rename (move) a file

Setting up a UNIX session
  • Go to the main WWW menu page from the HGMP homepage, using your favorite web browser.
  • Select “Unix” from the left-hand side (Tools menu), and then “Run Unix Now”.
  • Most of the services at the HGMP are only accessible for registered users. The first time you start up such a service, you will be presented with a dialog window, asking you to fill in your username and password.

The correct display name should be provided to you by your local computer person, but is usually something in the style of:

  • Where machine_name is the name of your local computer.
  • Click on the START UNIX button of the type of service you want to use. If you don’t have a X-WINDOWS client, there are alternatives like VNC and even JAVA, but they are quite slow.
  • A window running a simple Unix session will start. You will see the UNIX prompt:
Unix %
  • After this prompt you can start typing your UNIX commands. In the rest of the course, whenever you see the
Unix %
  • prompt, you can type in a UNIX command. Don’t type in the Unix % itself; only the command after the prompt needs to be typed.
Listing files
  • Give the command 'ls' to see the files that are currently in your directory:
Unix % ls
  • lots of filenames are shown, sorted on name and shown as 4 columns on a row.
Unix % ls acwords
  • the filename 'acwords' is displayed, showing you that the file exists
Unix % ls -l
  • the files are listed in much more detail, one file per line, showing the permissions , ownership, date of alteration, and name of the files
Unix % ls -l acwords
-rw-r----- 1 mrna11 1123 Mar 21 8:34 acwords
  • (permissions / owner / size(bytes) / alter-date / name)
Displaying content of files
  • Now try displaying the contents of files:
Unix % more seqn.seq
seqn.seq ..
cggagacggtagc.................tttgattcctgc
  • That was a small file. Now try displaying the contents of a larger file, like 'acwords':
Unix % more acwords
  • When a file is longer than a single page in length, the command more will stop at the end of each page and ask if you want to see more of it. It waits for one of several keys to be pressed:
  • To display the next page, press the space-bar.
  • To go back, press the 'b' key.
  • To see other commands you can give, press the '?' key.
  • DO NOT PRESS THE 'v' KEY - it gets you into a editor. (If you did get into the editor, press the colon ‘:’, ‘q’ and <ENTER> to get out)
  • To get out of more, press the 'q' key.
  • When you are tired of looking at words starting with 'ac', press 'q'
Copying Files
  • Now list the files with the command 'ls'. Note that there is no file with the name 'seqbn.seq'
  • Make a copy of the existing file 'seqn.seq' into the new file 'seqbn.seq'
Unix % cp seqn.seq seqbn.seq
  • List the files again. Note that the new file 'seqbn.seq' is now there.
  • Use 'more' to see the contents of the file 'seqbn.seq'. Note that it is identical to the contents of the original file 'seqn.seq'.
  • Copy the file 'seqn.seq' to a file 'seqcn.seq'.
  • List the files. See that you now have a collection of files starting with the name 'seq'
Renaming Files
  • Now rename the file 'seqn.seq' to the new name of 'seqan.seq'
  • Unix % mv seqn.seq seqan.seq
  • List the files. You will see that there is now no file called 'seqn.seq'. You were probably getting sick of this file anyway.
  • You will see that there is a new file called 'seqan.seq'. Display its contents using more. It is the same contents as 'seqn.seq' but known by a new name.
Deleting Files
  • Get rid of 'seqan.seq'. Remove it. Delete it. You do not want it anymore. You never want to see it again!
Unix % rm seqan.seq
  • List the files. There is no file called 'seqan.seq'!
  • Cautiously delete the file 'seqbn.seq' Use the command-line -i switch for the 'rm' command which causes 'rm' to ask if you are sure you want to delete the file:
Unix % rm -i seqbn.seq
rm: remove seqbn.seq (y/n)? y

3.2 FILES AND SUBDIRECTORIES

Directories are groups of files. They are organised in a hierarchy, so directories can have sub-directories and so on. You can think of directories as rooms in a house. You can only be in one room (directory) at a time. When you are in a room you can see everything in that room easily. To see things in other rooms, you have to go to the appropriate door and crane your head around.
To separate the names of directories and files, a slash '/' is used. If you have a set of directories: 
           HOME
            / \
           /   \
         prot  mysub

then you can be in your "HOME' directory and copy a file 'fred.dat' from the 'prot' directory to the 'mysub' directory:
Unix % cp prot/fred.txt mysub
You can be in the 'prot' directory and do the same thing:
Unix % cd prot
Unix % cp fred.txt ../mysub
Or you can be in the 'mysub' directory and do the same thing:
Unix % cd
Unix % cd mysub
Unix % cp ../prot/fred.txt .
In each case, you have to follow the path from the directory you are in to the file you want to copy and follow the path from the directory you are in to the directory you want to copy the file into. The directory you are currently in is indicated by a dot '.' in the last example.
  • Directories can be created, deleted and renamed.
  • Files can be moved or copied into specified directories.
You can only work in one directory at a time. This is known as your 'current working directory'. The directory that you start off in when you login is your 'home' directory. You can easily return to your home directory by giving the command 'cd'.
SOME MORE COMMANDS:
  1. mkdir make a new subdirectory
  2. cd change current directory (default is to home)
  3. cd .. move back up to the directory above this one
  4. pwd print current (working) directory
  5. mv file dir move a file into a directory
  6. rmdir remove (delete) directory (that is empty)

Making a Subdirectory
  • Make a new subdirectory called 'genome'
Unix % mkdir genome
  • List the files in your home directory, you will notice that there is a file called 'genome' there. A directory is a special sort of file. To see the difference, use the '-F' switch on 'ls', i.e. 'ls -F'. Directories have '/' after them.
  • Copy a file into the subdirectory 'genome'
Unix % cp acwords genome
  • List the files in the subdirectory 'genome'
Unix % ls genome
  • Display the file in the subdirectory 'genome' from the home directory above it
Unix % more genome/acwords
Deleting a Subdirectory
  • Attempt to remove the subdirectory 'genome'
Unix % rmdir genome
  • and you get the message:
rmdir: genome: Directory not empty
  • To remove a directory, you need to empty it of all its files first...
Changing Directories
  • Change directory - go down into the subdirectory 'genome'
Unix % cd genome
  • List the files in this directory. You will not see any of the files in the home directory above this one.
  • Check to see which directory you are in
Unix % pwd
/people/mrna11/genome
  • Change directory back to your home directory
Unix % cd
  • Double-check that you are in your home directory by giving the 'pwd' command and the 'ls' command. Do you recognise some of these files?
  • The directory above this one is indicated by two dots '..' So to list the files in the directory above the one you are in, give the command:
Unix % ls ..
Referring to files in directories
  • A file in a directory can be indicated by giving the directory name, then a slash '/' then the file name.
  • You already know how to copy the file 'acwords' into the directory genome:
Unix % cp acwords genome
  • But you can also copy 'acwords' into the directory 'genome' with a name other than 'acwords":
Unix % cp acwords genome/acw
A “thought” experiment (THINK...Don’t type!)
  • You can now combine several ideas. If you think you have followed things up until now, see if you can work out (without typing at your terminal) what is being done here. (It is a bit contrived...)
Unix % cd
Unix % mkdir genome/project
Unix % cd genome/project
Unix % cp ../../acwords thisac
Unix % cd ..
Unix % ls -l project
Unix % mv project/this* ..

3.3 EXTRA TRICKS IN UNIX

Here are some extra tricks with UNIX. You do not have to do these if you got left behind in this practical, but you may wish to know them.
Extra Unix short cuts
  • You can reduce the amount of typing of filenames with judicious use of the <TAB> key
Unix % ls geno<TAB>
  • The rest of the filename is filled in for you (if the filename exists, if not it beeps at you ). If the part of the file you typed in is not unique, it will beep at you and you can see the files that match what you typed in with < CTRL-D>
  • Also note that you can use the < UP> and < DOWN> arrow keys to recall previous commands, and the < LEFT> and < RIGHT> arrows to edit commands.. This is very useful!
Wildcards
  • Filenames can be wildcarded - you can use an asterisk '*' to replace part of a filename
Unix % ls alu*
Unix % more *.seq
UNIX aliases at the HGMP:
  • We have been teaching you the formal UNIX names of commands, but we have created friendly aliases for these commands (only available at the HGMP!) - enjoy:
Original HGMP Alias
  • ls dir
  • rm del, delete
  • cp copy
  • mv ren, rename
  • more ty, type
  • emacs ed, edit
  • pwd .
  • ls -alF v (verbose listing)

3.6 EDITORS

We use emacs (really MicroEmacs) and pico by default. You may prefer using any other editors on the HGMP system, for example:
  • vi - the old traditional UNIX editor, do not try this unless you know it already.
  • xedit - simple point-and-click editor for X-Windows users.
  • nedit – a very powerful point-and-click editor for X-Windows users.
  • pico - the editor used in the pine mail program. Lots of help in this editor.
Pico is a simple editor that always displays the available commands on the screen. It is the editor used inside the Pine mail program when editing a message, so you may prefer to use this. Whatever it would be useful for you to learn the basics of either Pico or emacs.
Editing a file with Pico
Unix % pico junk.txt
  • The editing commands are listed in the program.
  • e.g. To save a file and get out of pico, type <CTRL-X>.
Editing a file with Emacs
  • Start editing a file - the name of the file can be anything you like, e.g.:
Unix % emacs junk.txt
  • Start typing your life history. When you get to the end of a line, notice that it doesn't start a new line. You can type a line as long as you like.
  • When you press <RETURN> you will start a new line.
  • Save what you have typed with <ESC> Z. That is two key-presses, the <ESC> key and the 'Z' key.
  • You will be using the emacs editor later on in the course so if nothing else, remember how to save your file.
  • Start editing the file again.
Unix % emacs junk.txt
  • This time you will see that it has the contents that you have just typed in, instead of a blank screen.
  • Move the cursor around with the arrow keys. Where the cursor is placed, you can insert new characters - try this. You can also delete characters with the <DELETE> key - try this.
  • IF YOU EVER GET INTO TROUBLE WHEN USING EMACS, YOU CAN ABORT THE CURRENT COMMAND WITH <CTRL-G>
  • Some of the more interesting commands are:
<ESC> Q Reformat the current paragraph
<ESC> S something <ESC> Search forwards for 'something' in the text
<CTRL-K> Kill a line (delete) it
<CTRL-Y> Yank back the thing you just deleted (undelete it)

3.7 FILE TRANSFER PROTOCOL (FTP)

The files on the HGMP account are fysically located in Hinxton, UK and this is usually not where you yourself are located. More important, this is not where your printers are located. So if you wanted to print the contents of one of the files at your HGMP account, you could not just say print, or something equivalent, since then the file would be printed at the HGMP in Hinxton. Similarly, if you want to use the content of one of your files at the HGMP in the paper you are writing on your local PC, you need to transfer the files to your local computer. It also feasible that you have a data file on your local computer, that you wanted to do some analysis on using your HGMP account. In that case the file needs to be transfered from your local machine to the HGMP account.
Files can be transfered from one computer to the next in several ways, but all of them use the File Transfer Protocol or FTP. On unix machines transfering files can be done with the program FTP. It’s a command line program with a few simple commands. There are also graphical versions of FTP.
On PC’s and Mac’s there are several FTP programs, like FTP, Anarchy and Fetch. At the HGMP we also provide FTP access using the WWW.
Information: ftp from the Command Line
  • From a Unix command line you could enter:
Unix % ftp ftp.hgmp.mrc.ac.uk
  • Publicly available files can be accessed by using the user name anonymous, and entering your email as the password
  • To access your own files you would need to use your own user name and password. Commands available from the ftp > prompt can be obtained by typing “?”. The most commonly used commands are:
  • cd change directory
  • dir show files in directory
  • get transfer file from remote ftp site to local disk
  • put transfer file from local disk to remote ftp site
  • bye close connection and quit ftp
  • ascii switch to ascii (text) file transfer mode
  • binary switch to binary (executable) file transfer mode
ftp using a www Browser
  • It is usually easiest to use a www browser for retrieving files, e.g. type the following in the browser’s location window
ftp://ftp.ebi.ac.uk/pub/databases/
  • You get a list of the available files. Click a file name to save it to your local disk.
  • Note that this form of transfer only works to copy files by ftp from a remote computer running an ftp server.
Downloading a file using the Filemanager
  • Another way of using the www browser to display and download files is by using the WWW Filemanager.
  • From the main WWW Menu page click on the Files link, on the left side of the menu, under the heading Tools.
  • Click on Run Filemanager Now! to run the filemanager.
  • A browser window will appear, with a listing of the files and directories in your homedirectory. You can click on a file to show it contents.
  • Click on acwords to see the acwords file again.
  • If you want to save a file that is on the HGMP account, to your local computer (the one you are physically looking at), you can save the file, by using the Save as.. option in the File menu of the browser (NOT on the page, but in the menubar of your browser).
  • Save the file as a Text file (as opposed to the HTML option) to the local directory of your choice.
  • If you click on a directory name (like chapter4), you will go to a new page showing you the contents of that directory.
  • There are several other options you can try, like looking at the permissions and making a new directory. You can even delete files using the Filemanager! Clinking on the magnifying glass will give you those extra options.
Uploading a file using a www Browser
  • If you have a file on your local machine that you want to work with on your HGMP account, you have to transfer it to the HGMP, or UPLOAD it, in compu-jargon.
  • At the HGMP we have a few methods of uploading a file.
  • You can directly transfer a file from your local machine to your HGMP account, by using the program UPLOAD. It can be reached by choosing from the HGMP WWW menu, Utilities->Transfer Files->Easy file transfer by WWW upload. In the first field you can type in the name of the file on your local machine, or choose it by clicking the BROWSE button. If you want to keep the same name at your HGMP account, you can leave the second box blank, otherwise type in a different name. By clicking the UPLOAD FILE button, the file will be transfered.
  • Another way of getting a file uploaded to your HGMP account, is to paste it into a web-form. From the HGMP WWW menu choose, Utilities->Transfer Files->Cut & Paste data to transfer it to the HGMP, after clicking Run CUTPASTE NOW , copy your file content to the clipboard, using your local editor (MS-WORD for instance), and paste it into the box on the web-form. Type in a name under which this file is to be known at your HGMP account in the second box and click TRANSFER DATA .
